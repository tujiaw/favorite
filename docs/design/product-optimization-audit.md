# 产品优化审计

更新日期：2026-06-21

## 审计原则

- 优先优化高频路径：创建收藏、查找收藏、编辑内容、AI 辅助、设置模型。
- 优先减少摩擦：少切换页面、少重复点击、减少重复保存和失败后才提示。
- 优先提升可维护性：把 AI 能力拆成清晰模块，避免继续堆在 `App.tsx`。
- 每轮优化都必须通过 `npm run typecheck` 和 `npm run build`。

## 已落地

### AI 对话

- 改为右下角非模态浮窗，不再替换详情区。
- 支持最小化、关闭、拖动和窗口尺寸变化后的边界保护。
- 支持 `Ctrl/Cmd + Shift + J` 快捷唤醒，打开后自动聚焦输入框。
- 模型未配置时在浮窗内前置提醒，并提供设置入口。
- 流式输出改为按浏览器帧刷新，减少长回复时的打字顿挫感。
- 对话历史、模型选择、当前收藏上下文、写回收藏能力已保留。

### 创建收藏

- 快速保存增加保存中状态，避免连点或快捷键重复提交造成重复收藏。
- 保存失败会显示错误状态，不再静默失败。
- 粘贴内容后提供类型、标题、域名和摘要预览，保存前能确认系统识别结果。
- 粘贴内容后可以在预览区直接修改保存标题；不填写时继续使用系统识别标题，避免 URL 收藏默认只保存成粗略域名。
- Bitwarden 导入在读取、解析加密和逐条保存阶段都有进度提示，避免大批量导入时误以为页面卡住。

### 设置页

- 设置弹窗改成内容区滚动、底部操作固定在弹窗内，减少移动端和长列表下的挤压。
- 提示词列表增加空状态，避免新增入口不明确。

### 安全和保险箱

- 保险箱主密码不再使用 `localStorage` 长期保存；改为当前浏览器会话缓存，并在启动时清理旧的 `favorite-vault` 本地缓存。
- 保险箱过期选项文案从“永不过期”改为“本次会话不过期”，避免用户误解主密码会跨浏览器重启保存。
- 大模型设置同步到 Supabase 时不再上传 API Key；模型列表、Base URL、提示词可同步，API Key 仅保存在当前浏览器本地。

### 详情页和列表

- 详情头部在窄屏下改为换行布局，降低标题、标签、类型和操作按钮互相挤压的概率。
- 账号详情增加保险箱未解锁/过期提示，解释为什么不能查看或复制密码。
- 账号解密失败时不再静默失败，详情页展示恢复说明并提供“重新解锁保险箱”入口；显示密码失败时不会错误切换可见状态。
- 收藏列表项使用 `React.memo`，减少大量收藏下父级状态变化带来的重复渲染。
- 收藏列表视图升级为 `@tanstack/react-virtual` 虚拟行渲染；搜索和筛选仍覆盖全量数据。
- 网格视图也升级为虚拟行渲染，按当前列表宽度自动使用一列或两列，移除手动“加载更多”路径。
- 筛选/搜索导致空结果时提供一键清除筛选路径，避免用户停留在空列表。

### 页面反馈

- 全局状态提示移动到底部居中，避免和右下角 AI 浮窗互相遮挡。

### 代码组织

- AI 对话浮窗壳层拆到 `src/components/ai-chat-launcher.tsx`。
- AI 对话面板拆到 `src/components/ai-chat-panel.tsx`。
- AI 聊天历史、模型去重、上下文构造、标题/标签解析拆到 `src/ai/chat.ts`。
- 大模型设置的表单解析、模型行增删、本地 API Key 合并和云端脱敏逻辑拆到 `src/ai/settings.ts`，减少 `App.tsx` 对 AI 设置细节的直接承担。
- Bitwarden 导入的数据解析、去重、字段映射和加密准备逻辑拆到 `src/app/bitwarden.ts`，`App.tsx` 只保留状态提示、保存和刷新流程。
- 账号详情、复制用户名/密码和保险箱重解锁提示拆到 `src/components/account-detail-panel.tsx`，让详情主面板只负责内容类型分发。
- 左侧导航和收藏列表卡片拆到 `src/components/sidebar.tsx`、`src/components/item-card.tsx`，减少 `app-layout.tsx` 的展示组件堆叠。
- App 层错误处理从 `catch (error: any)` 收紧为 `unknown` + `errorMessage`，避免后续异常对象格式变化导致状态提示二次报错。

### 构建包体

- Markdown 预览改为懒加载，`react-markdown` 相关逻辑从主入口拆出。
- 生产构建中主入口包从约 1.33MB 降到约 1.13MB；AI/语法高亮相关大 chunk 仍需继续拆分。
- AI 对话面板中的 Streamdown/代码高亮/数学公式/Mermaid 渲染改为二级懒加载，`ai-chat-panel` chunk 从约 585KB 降到约 67KB，重型回复渲染器拆到 `message-response` 按需加载。
- AI 回复渲染继续拆分为轻量 Markdown/CJK 与富渲染两层；普通回复的 `message-response` chunk 从约 519KB 降到约 0.46KB，代码块/公式/Mermaid 才加载约 494KB 的 `message-response-rich`。
- AI 回复富渲染继续按数学公式、Mermaid 图表拆成独立懒加载组件；聊天代码块改用 Streamdown 基础渲染，不再引入 Shiki 语言包。

## 待继续审计
