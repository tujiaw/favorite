# 产品优化审计

更新日期：2026-06-21

## 审计原则

- 优先优化高频路径：创建收藏、查找收藏、编辑内容、AI 辅助、设置模型。
- 优先减少摩擦：少切换页面、少重复点击、减少重复保存和失败后才提示。
- 优先提升可维护性：把 AI 能力拆成清晰模块，避免继续堆在 `App.tsx`。
- 每轮优化都必须通过 `npm run typecheck` 和 `npm run build`。

## 已落地

### AI 对话

- 改为右下角非模态浮窗，不再替换详情区。
- 支持最小化、关闭、拖动和窗口尺寸变化后的边界保护。
- 支持 `Ctrl/Cmd + Shift + J` 快捷唤醒，打开后自动聚焦输入框。
- 模型未配置时在浮窗内前置提醒，并提供设置入口。
- 流式输出改为按浏览器帧刷新，减少长回复时的打字顿挫感。
- 对话历史、模型选择、当前收藏上下文、写回收藏能力已保留。

### 创建收藏

- 快速保存增加保存中状态，避免连点或快捷键重复提交造成重复收藏。
- 保存失败会显示错误状态，不再静默失败。
- 粘贴内容后提供类型、标题、域名和摘要预览，保存前能确认系统识别结果。

### 设置页

- 设置弹窗改成内容区滚动、底部操作固定在弹窗内，减少移动端和长列表下的挤压。
- 提示词列表增加空状态，避免新增入口不明确。

### 详情页和列表

- 详情头部在窄屏下改为换行布局，降低标题、标签、类型和操作按钮互相挤压的概率。
- 账号详情增加保险箱未解锁/过期提示，解释为什么不能查看或复制密码。
- 收藏列表项使用 `React.memo`，减少大量收藏下父级状态变化带来的重复渲染。
- 收藏列表改为首批渲染 + 加载更多，搜索和筛选仍覆盖全量数据，降低大量收藏下的初始 DOM 压力。
- 筛选/搜索导致空结果时提供一键清除筛选路径，避免用户停留在空列表。

### 页面反馈

- 全局状态提示移动到底部居中，避免和右下角 AI 浮窗互相遮挡。

### 代码组织

- AI 对话浮窗壳层拆到 `src/components/ai-chat-launcher.tsx`。
- AI 对话面板拆到 `src/components/ai-chat-panel.tsx`。
- AI 聊天历史、模型去重、上下文构造、标题/标签解析拆到 `src/ai/chat.ts`。

### 构建包体

- Markdown 预览改为懒加载，`react-markdown` 相关逻辑从主入口拆出。
- 生产构建中主入口包从约 1.33MB 降到约 1.13MB；AI/语法高亮相关大 chunk 仍需继续拆分。

## 待继续审计

- 账号页：解密失败时是否需要更强的恢复引导。
- 创建页：是否需要在允许 CORS 或后端代理的情况下自动读取网页标题。
- 列表页：首批渲染策略上线后，是否还需要在万级数据下升级为真正虚拟列表。
- 构建包体：AI/语法高亮相关 chunk 仍较大，需要进一步拆包策略。
