# 个人收藏中心设计文档

## 1. 背景

浏览器收藏夹主要面向网址收藏，无法很好地保存文本片段、图片、账号信息、代码片段、备注等零散资料。用户真正需要的是一个轻量的个人信息收藏中心：打开网页后，能快速粘贴内容，自动归类，并且之后能快速搜索和复用。

本项目第一版采用 Web/PWA 形态，部署到 Vercel，使用 Supabase 提供登录、数据库和文件存储能力。

## 2. 产品目标

第一版目标是做一个个人自用的轻量收藏工具：

- 支持快速粘贴链接、文本、图片、代码、JSON 等内容。
- 自动识别内容类型并保存。
- 支持搜索、标签、收藏标记和类型筛选。
- 支持账号密码类收藏，但敏感字段必须端到端加密。
- 支持 GitHub OAuth 登录。
- 支持跨设备访问。

第一版不做：

- 全局快捷键。
- 自动监听剪贴板。
- 截图收藏。
- 浏览器扩展。
- 团队协作。
- 计费系统。
- 浏览器密码自动填充。

## 3. 核心体验

首页就是主工作台，不做营销页。

用户进入应用后，最重要的入口是一个“万能输入框”。

用户可以：

- 粘贴 URL，系统保存为链接收藏。
- 粘贴普通文字，系统保存为文本收藏。
- 粘贴代码或 JSON，系统保存为代码片段。
- 粘贴图片，系统上传图片并生成图片收藏。
- 手动创建账号记录，填写网站、用户名、密码和备注。

保存后，收藏立即出现在列表中。

## 4. 信息架构

每条收藏记录包含：

- `id`
- `user_id`
- `type`: `link | text | image | code | json | account`
- `title`
- `content`
- `source_url`
- `domain`
- `preview`
- `tags`
- `note`
- `favorite`
- `storage_path`
- `encrypted_secret`
- `created_at`
- `updated_at`
- `last_used_at`
- `use_count`

其中：

- 普通文本、链接、标题、标签、备注可用于搜索。
- 图片文件存入 Supabase Storage。
- 密码、恢复码、私密备注等敏感字段放入 `encrypted_secret`。
- `encrypted_secret` 只保存密文，Supabase 不保存明文密码。

## 5. 页面设计

### 5.1 登录页

支持：

- GitHub OAuth
- Google OAuth

登录后进入收藏中心。

### 5.2 收藏中心

主要区域：

- 顶部：万能输入框。
- 左侧或顶部筛选区：类型、标签、收藏。
- 中间：收藏列表。
- 右侧或弹窗：详情编辑面板。

收藏列表显示：

- 类型图标。
- 标题。
- 预览内容。
- 来源域名。
- 标签。
- 收藏状态。
- 创建时间或最近使用时间。

详情面板支持：

- 编辑标题。
- 编辑标签。
- 编辑备注。
- 修改收藏类型。
- 复制内容。
- 删除收藏。
- 对链接执行打开操作。
- 对账号记录执行解锁、查看、复制密码操作。

## 6. 内容识别规则

万能输入框保存时执行轻量识别：

- 以 `http://` 或 `https://` 开头：识别为 `link`。
- 可解析为 JSON：识别为 `json`。
- 命中常见代码特征：识别为 `code`。
- 图片粘贴或拖拽：识别为 `image`。
- 其他内容：识别为 `text`。

账号密码不做自动猜测，使用单独的账号记录表单创建，避免误判敏感信息。

## 7. 安全设计

保险箱使用独立主密码。

流程：

1. 用户首次创建账号记录时设置主密码。
2. 前端用主密码派生加密密钥。
3. 敏感字段在浏览器本地加密。
4. 密文保存到 Supabase。
5. 用户下次查看密码时需要重新解锁保险箱。

敏感字段包括：

- 密码。
- 恢复码。
- 密保答案。
- 私密备注。

非敏感字段可以明文保存：

- 网站标题。
- URL。
- 用户名。
- 标签。
- 普通备注。

复制密码后，前端应在短时间后尝试清空剪贴板中由本应用写入的密码内容。

## 8. 技术架构

### 8.1 前端

推荐：

- Next.js
- React
- TypeScript
- Tailwind CSS 或 shadcn/ui
- PWA 支持

部署：

- Vercel

### 8.2 后端服务

使用 Supabase：

- Supabase Auth：GitHub/Google OAuth。
- Supabase Postgres：收藏元数据。
- Supabase Storage：图片和附件。
- Supabase RLS：按 `user_id` 做数据隔离。

### 8.3 数据权限

所有收藏记录必须绑定 `user_id`。

RLS 规则：

- 用户只能读取自己的收藏。
- 用户只能新增自己的收藏。
- 用户只能更新自己的收藏。
- 用户只能删除自己的收藏。
- Storage 文件路径应按用户隔离，例如：`{user_id}/{item_id}/image.png`。

## 11. 验收标准

### 普通收藏

- 用户可以粘贴 URL 并保存为链接收藏。
- 用户可以粘贴文本并保存为文本收藏。
- 用户可以粘贴图片并保存为图片收藏。
- 用户可以搜索标题、内容、标签和备注。
- 用户可以编辑、删除、收藏和取消收藏记录。

### 账号保险箱

- 用户可以创建账号记录。
- 未解锁时不能查看密码。
- 解锁后可以查看和复制密码。
- Supabase 中不能出现明文密码。
- 主密码错误时无法解密。

### 权限安全

- 用户只能看到自己的收藏。
- 未登录用户不能访问收藏中心。
- 不同用户之间数据隔离。
